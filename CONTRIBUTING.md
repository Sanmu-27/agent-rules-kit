# Contributing

Thanks for helping improve AI coding agent workflows.

## What Makes A Good Rule

A good rule is:

- actionable
- concise
- tool-friendly
- based on a real failure mode
- useful across many repositories

Prefer:

```text
Before editing a shared utility, search for all call sites and update tests that cover changed behavior.
```

Avoid:

```text
Write good code.
```

## Contribution Types

You can contribute:

- tool-specific instruction files
- domain-specific rules
- review prompts
- testing workflows
- examples from real agent sessions
- translations

## Style

- Use plain English.
- Keep rules short.
- Avoid hype.
- Do not include secrets, private code, or proprietary snippets.
- Prefer examples that can be copied directly.

## Pull Request Checklist

- The new rule has a clear use case.
- The rule does not conflict with existing guidance.
- The wording is specific and practical.
- Any tool-specific behavior is documented in the right folder.
