# Launch Checklist

Use this checklist before publishing the repository.

## Repository

- Choose a short repo name.
- Add a clear one-line GitHub description.
- Add topics: `ai`, `agents`, `coding-agent`, `codex`, `claude-code`, `cursor`, `prompts`.
- Add a social preview image.
- Pin a short demo in the README.

## README

- Show the problem in the first screen.
- Show the CLI command in the first screen.
- Include copy-paste examples.
- Include a table of supported tools.
- Explain how to contribute new rules.

## Release

- Create the first tag: `v0.1.0`.
- Publish to npm if the package name is available.
- Post on X, Hacker News, Reddit, LinkedIn, and relevant Discords.
- Ask for concrete feedback, not stars.

## Growth

- Add rules from real failures.
- Keep examples short and copy-ready.
- Accept community templates quickly.
- Add integrations only after the core rules are useful.
